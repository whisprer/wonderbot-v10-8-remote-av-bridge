3. format + mount the 1.92TB weights drive

7. the bit nearly everyone forgets — NUMA
Dual-socket Xeon = two NUMA nodes. Each P40 hangs off PCIe lanes owned by one socket. If inference threads land on the wrong socket relative to the GPU they're feeding, you eat cross-socket QPI/UPI latency for free. Check the topology:

bash
nvidia-smi topo -m        # shows which CPU affinity / NUMA node each GPU prefers
lscpu | grep NUMA
numactl --hardware
When you launch CPU-involved work, pin it to the GPU's local node, e.g.:

bash
numactl --cpunodebind=0 --membind=0  <your inference command>
For pure all-on-GPU inference this matters less, but for prompt-processing / CPU offload / KV-cache shuffling it's real throughput. Worth measuring once and baking the right numactl (or systemd CPUAffinity=) into yer service unit.

Also drop swappiness so the kernel never tries to page model-adjacent memory to disk:

bash
echo 'vm.swappiness=10' | sudo tee /etc/sysctl.d/99-llm.conf
sudo sysctl --system
